Giới thiệu hệ thống

Crawler Tool là hệ thống thu thập dữ liệu sản phẩm tự động từ các website thương mại điện tử. Hệ thống được phát triển với mục tiêu cung cấp giải pháp thu thập dữ liệu hiệu quả, dễ sử dụng và có thể mở rộng.

Hệ thống hỗ trợ thu thập dữ liệu từ nhiều website khác nhau với giao diện web thân thiện, cho phép người dùng dễ dàng cấu hình, theo dõi và quản lý quá trình thu thập dữ liệu.

Tính năng chính
  • Thu thập dữ liệu sản phẩm tự động
  • Quản lý cấu hình website
  • Xuất dữ liệu định dạng CSV/JSON
  • Theo dõi lịch sử thu thập
  • Quản lý bookmark và từ khóa
  • Giao diện web hiện đại
Lợi ích
  • Tiết kiệm thời gian thu thập dữ liệu
  • Tự động hóa quy trình thu thập
  • Hỗ trợ nhiều website cùng lúc
  • Dữ liệu được cấu trúc và sạch
  • Giao diện trực quan, dễ sử dụng
  • Có thể mở rộng và tùy chỉnh

Kiến trúc hệ thống

Hệ thống Crawler Tool được thiết kế theo kiến trúc client-server với sự kết hợp giữa Node.js và Python để tối ưu hóa hiệu suất và khả năng mở rộng.

Cấu trúc dự án

  • html/ - Các trang web
  • assets/ - Tài nguyên tĩnh
  • assets/js/ - JavaScript modules
  • assets/css/ - Stylesheets
  • assets/img/ - Hình ảnh

  • js/server.js - Express server
  • js/config.js - Cấu hình hệ thống
  • js/data/ - Dữ liệu JSON
  • js/helpers.js - Utility functions

  • crawv2/ - Python crawler v2
  • crawv2/crawlv2_simple.py - Main crawler
  • crawv2/websites_config.json - Cấu hình
  • crawv2/downloads/ - File kết quả

Hướng dẫn thu thập dữ liệu

Hệ thống cung cấp hai phương thức thu thập dữ liệu: Thu thập cơ bản và Thu thập v2 với nhiều tính năng nâng cao.

Mô tả:

Phương thức thu thập dữ liệu cơ bản sử dụng API có sẵn của các website thương mại điện tử. Phù hợp cho các website hỗ trợ WooCommerce API.

Hướng dẫn thực hiện:
  1. Truy cập trang "Thu thập dữ liệu" từ menu chính
  2. Chọn website từ danh sách có sẵn trong dropdown
  3. Nhập số trang cần thu thập (mặc định: 10 trang)
  4. Nhấn nút "Bắt đầu thu thập"
  5. Theo dõi tiến trình trong bảng thống kê real-time
  6. Chờ quá trình hoàn tất và kiểm tra kết quả
Lưu ý quan trọng:
  • Chỉ hoạt động với các website hỗ trợ WooCommerce API
  • Tốc độ thu thập phụ thuộc vào API của website đích
  • Dữ liệu được lưu tự động trong hệ thống

Mô tả:

Phương thức thu thập dữ liệu v2 sử dụng web scraping với Python, cho phép thu thập từ bất kỳ website nào. Hỗ trợ cấu hình linh hoạt và tính năng bookmark link.

Bước 1: Tạo cấu hình thu thập
  1. Truy cập trang "Thu thập dữ liệu v2"
  2. Nhấn nút "Thêm cấu hình"
  3. Điền đầy đủ thông tin cấu hình:
    • Tên cấu hình: Tên dễ nhớ để quản lý
    • URL gốc: URL chính của website
    • Base URL crawl: URL trang sản phẩm
    • Page URL template: Template URL với {page} placeholder
    • Image selector: CSS selector để tìm hình ảnh
    • Bookmark link: URL đánh dấu (tùy chọn)
  4. Nhấn "Thêm cấu hình" để lưu
Bước 2: Bắt đầu thu thập
  1. Chọn cấu hình từ danh sách
  2. Nhập trang bắt đầu (mặc định: 1)
  3. Nhập trang kết thúc (mặc định: 10)
  4. Chọn "Lưu dữ liệu real-time" nếu muốn lưu ngay
  5. Nhấn "Bắt đầu" để khởi động
Tính năng đặc biệt:
  • Bookmark Link: Hệ thống sẽ hỏi có muốn tiếp tục khi gặp URL đánh dấu
  • Real-time Monitoring: Theo dõi tiến trình thu thập theo thời gian thực
  • Error Handling: Tự động xử lý lỗi và thông báo cho người dùng
  • Flexible Configuration: Có thể cấu hình cho bất kỳ website nào

Xử lý sự cố thường gặp

Hướng dẫn khắc phục các lỗi thường gặp trong quá trình sử dụng hệ thống.

Nguyên nhân:
  • URL website không chính xác
  • Website đã thay đổi cấu trúc
  • Website tạm thời không khả dụng
Giải pháp:
  1. Kiểm tra lại URL trong cấu hình
  2. Thử truy cập website trực tiếp trên trình duyệt
  3. Cập nhật lại cấu hình nếu cần thiết
  4. Thử lại sau vài phút nếu website tạm thời lỗi

Nguyên nhân:
  • Website phản hồi chậm
  • Kết nối mạng không ổn định
  • Website có bảo vệ chống bot
Giải pháp:
  1. Kiểm tra kết nối mạng
  2. Thử thu thập ít trang hơn
  3. Thêm delay giữa các request
  4. Kiểm tra xem website có chặn bot không

Nguyên nhân:
  • CSS selector không chính xác
  • Website sử dụng JavaScript để load nội dung
  • Cấu trúc HTML đã thay đổi
Giải pháp:
  1. Kiểm tra lại CSS selector trong cấu hình
  2. Sử dụng Developer Tools để kiểm tra HTML
  3. Cập nhật selector phù hợp với cấu trúc mới
  4. Thử các selector khác nhau

Nguyên nhân:
  • URL bookmark không chính xác
  • Server Node.js không chạy
  • Lỗi kết nối giữa Python và Node.js
Giải pháp:
  1. Kiểm tra URL bookmark trong cấu hình
  2. Đảm bảo server Node.js đang chạy
  3. Kiểm tra kết nối localhost:3000
  4. Restart server nếu cần thiết
Mẹo tối ưu hóa
Để tăng tốc độ thu thập:
  • Giảm số trang thu thập mỗi lần
  • Sử dụng thu thập cơ bản khi có thể
  • Đảm bảo kết nối mạng ổn định
  • Chọn thời gian ít người dùng
Để tăng độ chính xác:
  • Kiểm tra CSS selector kỹ lưỡng
  • Test cấu hình với ít trang trước
  • Cập nhật cấu hình thường xuyên
  • Sử dụng bookmark link để kiểm soát
